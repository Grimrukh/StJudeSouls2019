NOTES:
- Including all logic grouping brackets is ugly, so I have disabled them by default. It is
generally safe to assume that logical operations evaluate from left to right when they are all
one type, and that later OR operations are evaluated before earlier AND operations. Set
`full_brackets=True` for explicit order.
- &: values that have been previously computed in the current condition evaluation for this state
and loaded from registers.
- ^: interpreter should continue even if the previous value is false.
- !: interpreter should stop if the previous value is false. (Yes, this is not logically consistent
with the above, but I'm not certain exactly what makes the interpreter halt during a line. It may
halt whenever a zero value is not saved to a register, hence why this 'null register' is used.)
